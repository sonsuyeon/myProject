1.1 콘텐츠에 의미 정의 및 목적부여

< 컴포넌트를 사용하는 이유 >

- 상단 카테고리 영역 오신작(최신등록작품)/로맨스/판타지/BL 클릭할 때,
categoryChange 함수를 실행시켜 아래 작품리스트가 일치하는 카테고리 리스트를 불러옵니다.

- CategoryList.vue 라는 구조를 동일하게 사용하여 중복코드 사용을 줄였습니다.

`<section>` `<article>`의 차이점 및 div 사용 지양이 의미하는 바

시멘틱마크업을 하면서 제일 고민한 부분이 section태그를 쓸 것인지, article 태그를 쓸 것인지에 대한 것이었습니다.

1. 의미와 목적

`<section>` : 주제별로 콘텐츠를 구분하고 페이지를 구조화하는 데 사용됨.

`<article>` : 독립적이고 자체적으로 의미를 가지는 콘텐츠를 나타냄. 문서나 페이지의 다른 부분과 관계없이도 충분히 이해될 수 있어야 함.

2. 재사용 가능성

`<section>` : 문서의 한 부분으로서, 그 자체로 완전한 의미를 가지지 않습니다. 문맥이 필요하며, 다른 섹션들과 결합해 더 큰 의미를 형성합니다.

`<article>` : 독립적으로 재사용이 가능하며, 다른 문서나 페이지로 이동하더라도 그 의미가 변하지 않습니다.

3. 내부 구조

`<section>` : 보통 다른 요소들(예: `<header>`, `<h2>`, `<p>`, `<article>`)을 포함할 수 있으며, 특정 주제에 대한 설명이나 정보를 제공합니다.

`<article>` : 자체적으로 완전한 콘텐츠이며, 제목과 본문을 포함하는 것이 일반적입니다.

두 태그의 차이점을 찾아보고 예시를 계속 고민하였고, 정답은 계속해서 사용해보는 것 밖에 없다고 생각하여 매 이벤트, 기획전을 작업할 때 기획서와 디자인 요소 확인 후 시멘틱 태그를 먼저 고려하여 작업하였습니다.

2. SCSS 도입

2.1 BEM 방법론 적용

  • BEM 방법을 통해 작성자가 아닌 다른 작업자가 코드를 보아도 목적과 기능을 직관적으로 파악 가능
  • CSS만으로 구조 파악 가능
  • 높은 재사용성

코멘트 영역에 .wrk-comment__list / .wrk-comment__item / wrk-comment__btn--more 등의 클래스로 작성하였으며,
wrk-comment__item--none에서 wrk-comment는 Block,item는 Element, none은 modifier가 됩니다.

wrk-comment는 작품의 댓글을 의미하며, item요소가 none(없는) 상태를 나타내는 작품댓글이 없음을 의미합니다.

CategoryList.vue 라는 구조를 동일하게 사용하여 중복코드 사용을 줄였습니다.

2.2 SCSS 기법 활용

글자수가 넘치게 되어 말줄임표가 필요한 경우
@mixin - @include 기법을 활용해 중복되는 코드를 쉽게 불러오며, 수정이 있을 경우 한 번에 변경할 수 있도록 했습니다.
별도로 css를 수정해주어야 할 때는 @include 아래 -webkit-line-clamp: 2; 작성해준 것처럼 추가했습니다.
