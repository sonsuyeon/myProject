1. 상단 탭메뉴 네비게이션 고정

스크롤 위치에 따라 탭 메뉴를 활성화하여 특정 섹션에 도달하게 되면 네비게이션을 고정해줌으로써 사용자 경험을 개선하는데 이점이 있습니다.

1. 사용자 편의성 증대
사용자가 원하는 섹션으로 쉽게 이동할 수 있으며, 페이지를 탐색할 때 필요한 메뉴를 찾기위해 다시 스크롤을 올릴 필요가 없어져 사용자 경험이 향상됩니다.
2. 가시성 향상
3. 집중력 유지

- 필요한 정보에 집중할 수 있습니다.

4. 디자인 일관성

- 일관된 레이아웃을 가지게 되어 사용자가 페이지를 탐색할 때 안정감을 느끼게 하며, 웹사이트의 신뢰성에도 긍정적인 영향을 미칠 수 있습니다.

5. SEO 및 마케팅 측면

- 페이지 체류 시간 증가로 방문자의 이벤트 참여율을 높일 수 있습니다.

1.1 요소 찾기

- 이벤트 페이지로 분류시킨 jestina-detail__event 클래스가 있는 요소인지 체크하고 그 페이지에 `tabContent2024` 로 특정 콘텐츠가 포함되어 있을 때만 실행시키도록 합니다.

- const firstSection = document.querySelector('.jestina-detail__event-wrap');
⇒ 네비게이션 바가 화면 상단에 고정될 시점을 나타내줍니다.

1.2 위치계산

＊ 현재 스크롤 위치 확인

해당 페이지를 의미하는 window객체와 그 객체의 위치 값을 읽는 scrollY의 속성 값을 활용해 현재 스크롤이 어디까지 내려갔는지 확인 할 수 있습니다.

＊ 섹션의 높이와 고정된 헤더 및 네비게이션 바의 높이 계산

1.3 비활성화시키기

스크롤 후 여러 개의 탭에 `actvie` 클래스가 남아 있으면, 사용자 입장에서 여러개의 탭이 활성화된 것처럼 보일 수 있습니다. 이것은 현재 사용자가 보고 있는 섹션에 대한 정보를 명확하게 전달하지 못하게 되므로, 하나의 탭만 활성화가 되어야하므로 모든 탭메뉴에서 `active` 클래스를 제거하여 비활성화해 초기화를 시켜줍니다.

1.4 스크롤 위치에 따라 활성화 시킬 탭 결정

if문을 사용하여 조건으로 체크하고 스크롤 위치에 따라 적절한 탭에 'active' 클래스를 추가하여 활성화합니다.

  • 섹션이 아래에 있을 경우 위에 있는 섹션을 따로 확인하는 불필요한 체크가 필요 없습니다.
  • 스크롤 위치가 경계에 있을 경우 적절한 섹션을 활성화시켜 경계에서 중복 활성화되는 것을 막아줍니다.

1.5 네비게이션 바 고정

＊ 네비게이션 바가 첫 번째 섹션에 진입하게 되면 상단에 고정시키도록 해줍니다.

2.2 모바일 헤더 이슈

모바일 헤더의 경우
스크롤을 올릴때 scrollUp 클래스가 들어가면서 헤더가 함께 고정되며,
스크롤을 내릴 때 scrollDown 클래스가 들어가면서 헤더 없이 고정이 되어야하는데
올릴 때 헤더영역이 탭메뉴를 덮어버리는 문제가 발생함.

PC MOBILE

script를 동일하게 사용하였지만 오류가 발생하였고 콘솔을 찍어 헤더 높이 값을 체크해본 결과,
PC와 MOBILE에서 헤더높이값으로 구하는 영역이 각각 다르기 때문에
모바일에서는 header 영역의 높이값이 구해지지 않았고 두 개가 겹쳐져 이슈가 발생한 것으로 확인되었습니다.
